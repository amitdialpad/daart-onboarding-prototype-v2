## üìù Documentation Standards

### File Naming
- Use snake_case for markdown files
- Be descriptive but concise
- Include date in filename for time-sensitive docs

### Content Structure
Each documentation file should include:
1. **Title & Purpose**: What this document covers
2. **Last Updated**: When it was last modified
3. **Key Changes**: Summary of changes (for implementation docs)
4. **Detailed Content**: The main content
5. **Related Docs**: Links to related documentation

### Keeping Docs Updated
- Update PROJECT_STATUS.md after major changes
- Add entries to relevant implementation guides
- Mark outdated docs with **[ARCHIVED]** in title
- Reference new features in README.md
